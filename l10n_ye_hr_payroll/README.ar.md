# التوطين اليمني للرواتب (l10n_ye_hr_payroll)

وحدة Odoo 16 متكاملة ومتوافقة تماماً مع **قانون العمل اليمني رقم 5 لعام 1995**، وأنظمة التأمينات الاجتماعية، وقانون ضريبة الدخل اليمني (ضريبة الأجور والمرتبات).

---

## الميزات

### 1. تقويمات العمل
- **التقويم اليمني القياسي (48 ساعة)**: السبت – الخميس، 8 ساعات/يوم، الجمعة إجازة
- **التقويم اليمني الإداري (40 ساعة)**: الأحد – الخميس، 8 ساعات/يوم، الجمعة والسبت إجازة
- ربط تلقائي مع العقود عبر نوع هيكل الراتب (كود الدولة YE)

### 2. أنواع إدخالات العمل
7 أنواع لتسجيل الحضور والإضافي والإجازات:

| الكود | النوع | الغرض |
|-------|-------|-------|
| `WORK100` | حضور | ساعات العمل القياسية |
| `OT_DAY` | إضافي | العمل الإضافي النهاري (1.5x) |
| `OT_NIGHT` | إضافي | العمل الإضافي الليلي (2.0x) |
| `LEAVE100` | إجازة | إجازة سنوية |
| `LEAVE110` | إجازة | إجازة مرضية |
| `LEAVE90` | إجازة | إجازة بدون راتب |
| `LEAVE120` | إجازة | إجازة عيد |

### 3. قواعد الراتب (13 قاعدة)

| # | الكود | الوصف | الإشارة |
|---|-------|-------|---------|
| 1 | `BASIC` | الراتب الأساسي من `contract.wage` | + |
| 2 | `HOUSING_ALW` | بدل سكن (ثابت، خاضع للتأمين) | + |
| 3 | `TRANSPORT_ALW` | بدل مواصلات (ثابت، خاضع للتأمين) | + |
| 4 | `OTHER_ALW` | بدلات متغيرة (غير خاضعة للتأمين) | + |
| 5 | `OT_DAY` | العمل الإضافي النهاري 1.5x | + |
| 6 | `OT_NIGHT` | العمل الإضافي الليلي 2.0x | + |
| 7 | `GROSS` | إجمالي الراتب = BASIC + ALW | + |
| 8 | `EMP_INS` | تأمين اجتماعي موظف 6% (بحدود دنيا/قصوى) | – |
| 9 | `TAX` | ضريبة الدخل التصاعدية (0%/10%/15%/20%) | – |
| 10 | `UNPAID_DED` | خصم الإجازة بدون راتب | – |
| 11 | `NET` | صافي الراتب المستحق | + |
| 12 | `COMPANY_INS` | تأمين اجتماعي صاحب العمل 9% (محاسبي فقط) | + |
| 13 | `EOS_PROV` | مخصص نهاية الخدمة (محاسبي فقط) | + |

### 4. التأمينات الاجتماعية (قابلة للتكوين)
- نسبة الموظف (افتراضي 6%) ونسبة صاحب العمل (افتراضي 9%)
- حد أدنى وحد أقصى للوعاء التأميني (قابل للتكوين)
- الوعاء التأميني = الراتب الأساسي + بدل السكن + بدل المواصلات
- تكوين الحسابات المحاسبية لكل شركة (الإعدادات → الرواتب)

### 5. ضريبة الدخل التصاعدية
- قابلة للتكوين عبر نموذج `ye.tax.bracket` المخصص
- الشرائح الافتراضية (حسب القانون اليمني):
  - 0% على أول 10,000 ريال (إعفاء)
  - 10% من 10,001 إلى 30,000 ريال
  - 15% من 30,001 إلى 60,000 ريال
  - 20% على ما يزيد عن 60,000 ريال
- الضريبة = الإجمالي – الإعفاء القانوني – التأمين الاجتماعي

### 6. مخصص نهاية الخدمة (EOS)
- صيغة قابلة للتكوين عبر نموذج `ye.eos.rule`
- الافتراضي: شهر واحد لكل سنة خدمة (يُستحق شهرياً كـ `الراتب / 12`)
- محاسبي فقط (لا يؤثر على صافي الراتب)

### 7. الإجازات الرسمية (العطل الثابتة)
عطل وطنية محملة مسبقاً:
- 1 مايو (عيد العمال)
- 22 مايو (عيد الوحدة)
- 26 سبتمبر (ثورة سبتمبر)
- 14 أكتوبر (ثورة أكتوبر)
- 30 نوفمبر (عيد الاستقلال)

> **ملاحظة**: أعياد الأضحى والفطر (هجرية متحركة) يجب إضافتها سنوياً من قبل مدير الموارد البشرية عبر *تقني → الموارد → إجازات التقويم*.

### 8. استحقاق الإجازة السنوية (30 يوماً/السنة)
دليل إعداد وظيفي لاستحقاق 2.5 يوم شهرياً عبر نظام خطة الاستحقاق في Odoo.

### 9. التكامل المحاسبي الكامل
- قيود يومية للرواتب مع حسابات مدينة/دائنة قابلة للتكوين لكل قاعدة
- سجلات مساهمة لهيئة التأمينات الاجتماعية ومصلحة الضرائب
- قواعد محاسبية فقط لتكاليف صاحب العمل (COMPANY_INS, EOS_PROV)

```bash
python odoo-bin -i l10n_ye_hr_payroll -d <اسم_قاعدة_البيانات> \
```

## الإعداد بعد التنصيب

### الخطوة 1: الحسابات المحاسبية
اذهب إلى **الرواتب → الإعدادات → الإعدادات** وقم بتكوين:
- حساب دائن للتأمينات الاجتماعية
- حساب مصروف للتأمينات الاجتماعية
- حساب دائن للضرائب
- حساب التزام مخصص نهاية الخدمة
- حساب مصروف مخصص نهاية الخدمة
- نسبة التأمين، الحدود الدنيا/القصوى، مقدار الإعفاء

### الخطوة 2: شرائح الضريبة
اذهب إلى **الرواتب → الإعدادات → شرائح الضريبة اليمنية** → مراجعة/تعديل الشرائح الأربعة الافتراضية.

### الخطوة 3: قواعد نهاية الخدمة
اذهب إلى **الرواتب → الإعدادات → قواعد نهاية الخدمة** → تعريف الصيغ (الافتراضي: شهر/سنة).

### الخطوة 4: حسابات قواعد الراتب
اذهب إلى **الرواتب → الإعدادات → قواعد الراتب** → لكل قاعدة، املأ تبويب المحاسبة:

| القاعدة | الحساب المدين | الحساب الدائن |
|---------|--------------|---------------|
| BASIC, HOUSING_ALW, TRANSPORT_ALW, OTHER_ALW, OT_DAY, OT_NIGHT, GROSS, NET | مصروف رواتب | رواتب مستحقة للموظف |
| EMP_INS | رواتب مستحقة للموظف | تأمينات اجتماعية مستحقة |
| TAX | رواتب مستحقة للموظف | ضرائب مستحقة |
| UNPAID_DED | رواتب مستحقة للموظف | مصروف رواتب (عكس) |
| COMPANY_INS | مصروف تأمينات | تأمينات اجتماعية مستحقة |
| EOS_PROV | مصروف نهاية خدمة | التزام نهاية خدمة |

### الخطوة 5: خطة الاستحقاق
اذهب إلى **الموظفين → الإعدادات → الإجازات → خطط الاستحقاق**:
- إنشاء خطة "إجازة سنوية (2.5 يوم/شهر)" ← مستوى: 2.5 يوم، شهري
- تعيين للموظفين عبر التخصيص (النوع = استحقاق)

### الخطوة 6: عقود الموظفين
افتح كل عقد واضبط: البدلات، رقم التأمين الاجتماعي، قاعدة نهاية الخدمة.

## سيناريوهات الاختبار

| السيناريو | الراتب | البدلات | إضافي | بدون راتب | صافي متوقع |
|-----------|--------|---------|-------|-----------|------------|
| أ (منخفض) | 9,000 | 0 | 0 | 0 | 9,000 (لا ضريبة، لا تأمين) |
| ب (متوسط) | 25,000 | 5,000 | 10 ساعة نهار | 0 | تأمين على 30k، ضريبة على (30k - 10k - 1.8k) = 18.2k × 10% = 1,820 |
| ج (مرتفع) | 80,000 | 15,000 | 0 | 2 يوم | تأمين على 95k (بحد أقصى)، ضريبة 20% فوق 50k |
